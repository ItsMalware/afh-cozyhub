# Notion Database Schema Guide

To run CozyHub in "Live Mode" connected to your own Notion workspace, you must create four databases and provide their IDs in your `.env.local` file. The app is highly flexible and will match on several different property names.

Here are the required and recommended properties for each database.

## 1. Businesses (or Projects/Areas)
This database tracks high-level groupings of your work, comparing planned time vs. actual time spent.
**Environment Variable:** `NOTION_DATABASE_BUSINESSES_ID`

| Property Type | Property Name (Any of these work) | Description |
| --- | --- | --- |
| `Title` | Name | The name of the business or project area. |
| `Number` | `Weekly Target (min)`, `Weekly Target`, `Planned Minutes` | Your weekly target focus time (in minutes). |
| `Number` / `Rollup` / `Formula` | `Minutes Logged`, `Focused Minutes`, `Actual Minutes` | Total minutes logged. Commonly a Rollup of completed Tasks or Sessions. |

## 2. Tasks (The Focus Queue)
This is your main to-do list that populates the CozyHub queue.
**Environment Variable:** `NOTION_DATABASE_TASKS_ID`

| Property Type | Property Name (Any of these work) | Description |
| --- | --- | --- |
| `Title` | Name / Title | The task description. |
| `Relation` | `Business` | Relation to the Businesses database. |
| `Status` / `Select` | `Status`, `Task Status`, `Queue Status` | Used to track "Queued", "Next", "Live", and "Done". |
| `Status` / `Select` | `Queue Label`, `Queue` | Used to tag tasks (e.g., "Founder"). Tagging with "Agent Inbox" will hide it from focus mode. |
| `Date` | `Due Date`, `Due`, `Deadline` | When the task is due. |
| `Number` / `Text` | `Planned Minutes`, `Minutes`, `Duration` | Estimated time to complete (in minutes). |
| `Rich Text` | `Category`, `Area`, `Workstream` | E.g., "Admin", "Ops", "Deep Work". |

## 3. Sessions (Focus Logs)
This database serves as an immutable log of every focus timer session you complete.
**Environment Variable:** `NOTION_DATABASE_SESSIONS_ID`

| Property Type | Property Name (Any of these work) | Description |
| --- | --- | --- |
| `Title` | Name / Title | E.g., "Focus Session" (auto-generated by the app). |
| `Relation` | `Task` | Relation to the Tasks database (what you worked on). |
| `Status` / `Select` | `Status`, `Session Status` | Tracks "Active", "Live", or "Done". |
| `Date` | `Date`, `Started At` | Timestamp of when the session started/occurred. |
| `Number` | `Duration (Minutes)`, `Actual Minutes`, `Mins` | The raw number of minutes focused. |
| `Rich Text` | `Summary`, `Notes`, `Agent Notes` | Free-form text written by you or the Prime agent detailing what happened. |

## 4. Projects (Optional / Meta)
Currently, `NOTION_DATABASE_PROJECTS_ID` is defined in `.env.example`. This database is used internally by the agent tools for higher-level planning if you have a multi-layered tier (Portfolios -> Projects -> Tasks). The schema varies depending on your preferred methodology, but linking Projects to Tasks is recommended.
